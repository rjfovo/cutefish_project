# Cutefish Qt6/KDE6 迁移黑屏问题修复总结

## 问题描述
在将 Cutefish 桌面从 Qt5/KDE5/Debian12 迁移到 Qt6/KDE6/Debian13 后，系统出现黑屏问题：
- 使用 `startx` 运行后只有黑屏，只有一个鼠标可以自由活动
- 任何程序都无法打开和显示

## 问题分析
通过分析系统环境配置，发现 `/etc/profile.d/cutefish-qt6.sh` 文件存在以下问题：

### 1. 语法错误
- 第一行：`!/bin/sh` 应该是 `#!/bin/sh`
- 缺少 shebang 符号 `#`，导致脚本无法正确执行

### 2. 逻辑错误导致 Qt 使用 offscreen 平台
原脚本中的逻辑：
```bash
# 无头环境支持
if [ -z "$DISPLAY" ] && [ -z "$WAYLAND_DISPLAY" ]; then
    echo "检测到无头环境，设置Qt使用offscreen平台"
    export QT_QPA_PLATFORM=offscreen
    export QT_DEBUG_PLUGINS=0
fi
```

问题分析：
- 在正常的桌面环境中，`DISPLAY` 和 `WAYLAND_DISPLAY` 环境变量由 X11/Wayland 会话管理器设置
- 脚本在检测到这些变量未设置时，错误地将 `QT_QPA_PLATFORM` 设置为 `offscreen`
- `offscreen` 平台是用于无头环境的虚拟显示平台，会导致应用程序无法显示窗口

### 3. QT_QPA_PLATFORM 设置问题
原设置：`export QT_QPA_PLATFORM=xcb;wayland`
- 使用分号分隔多个平台，但 Qt6 期望使用冒号分隔
- 应该明确指定主要平台，避免自动检测失败

## 修复方案

### 1. 修复语法错误
- 将 `!/bin/sh` 改为 `#!/bin/sh`

### 2. 修正环境变量逻辑
- 移除自动检测 `DISPLAY` 和 `WAYLAND_DISPLAY` 的逻辑
- 只在明确需要无头环境时才手动设置 `QT_QPA_PLATFORM=offscreen`
- 添加注释说明原因

### 3. 优化 Qt 平台设置
- 设置 `QT_QPA_PLATFORM=xcb` 作为默认平台
- 移除 `;wayland` 后缀，避免解析问题
- 添加 Qt 插件路径设置，确保 Qt6 插件能被正确找到

### 4. 修复后的脚本内容
```bash
#!/bin/sh
# Qt6环境变量设置
export QT_QPA_PLATFORMTHEME=cutefish
export QT_STYLE_OVERRIDE=cutefish
export QT_QPA_PLATFORM=xcb
export QT_AUTO_SCREEN_SCALE_FACTOR=1
export QT_QPA_GENERIC_PLUGINS=evdevtablet
export QT_WAYLAND_DISABLE_WINDOWDECORATION=1
export QT_QUICK_CONTROLS_STYLE=fish-style
export XDG_CURRENT_DESKTOP=Cutefish

# 无头环境支持 - 只有在明确需要offscreen时才设置
# 注意：不要自动检测DISPLAY和WAYLAND_DISPLAY，因为X11/Wayland会话管理器会设置这些
# 只有在明确运行在无头环境时才手动设置QT_QPA_PLATFORM=offscreen

# 设置Qt插件路径
if [ -d "/usr/lib/x86_64-linux-gnu/qt6/plugins" ]; then
    export QT_PLUGIN_PATH="/usr/lib/x86_64-linux-gnu/qt6/plugins:$QT_PLUGIN_PATH"
fi

if [ -d "/usr/lib/qt6/plugins" ]; then
    export QT_PLUGIN_PATH="/usr/lib/qt6/plugins:$QT_PLUGIN_PATH"
fi
```

## 总结
黑屏问题的根本原因是 `/etc/profile.d/cutefish-qt6.sh` 脚本中的逻辑错误导致 Qt 使用了 `offscreen` 平台。修复后，Qt 将正确使用 `xcb` 平台，应用程序能够正常显示窗口。

此修复解决了从 Qt5/KDE5 迁移到 Qt6/KDE6 过程中的显示问题，为后续的兼容性修复奠定了基础。
