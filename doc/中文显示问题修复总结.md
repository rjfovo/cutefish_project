# 中文显示问题修复总结

## 问题描述
在Qt6/KDE6迁移后，CutefishOS桌面环境中出现中文无法显示、程序字符乱码的问题。

## 问题分析
经过分析，发现主要问题在于Qt6中`QFontDatabase::systemFont()` API的变化：

1. **Qt6 API变化**：在Qt6中，`QFontDatabase::systemFont()` API返回空字体，而不是Qt5中的有效字体
2. **影响范围**：影响所有使用该系统字体API的组件，特别是设置应用中的字体管理模块
3. **具体表现**：
   - 系统设置中的字体选择界面无法正确显示当前系统字体
   - 应用程序界面中的中文字符显示为方框或乱码
   - 字体回退机制失效

## 修复方案

### 1. 修复字体API兼容性问题
修改文件：`cutefish/code/settings/src/fontsmodel.cpp`

**原代码：**
```cpp
QString FontsModel::systemGeneralFont() const
{
    return QFontDatabase::systemFont(QFontDatabase::GeneralFont).family();
}

QString FontsModel::systemFixedFont() const
{
    return QFontDatabase::systemFont(QFontDatabase::FixedFont).family();
}
```

**修复后代码：**
```cpp
QString FontsModel::systemGeneralFont() const
{
    // Qt6 compatibility: QFontDatabase::systemFont() may return empty font
    // Use QFont() constructor with default font family
    QFont font;
    QString family = font.family();
    if (family.isEmpty()) {
        // Fallback to default font
        family = "Noto Sans";
    }
    return family;
}

QString FontsModel::systemFixedFont() const
{
    // Qt6 compatibility
    QFont font("Monospace");
    QString family = font.family();
    if (family.isEmpty()) {
        // Fallback to default monospace font
        family = "Monospace";
    }
    return family;
}
```

### 2. 修复原理
- **Qt6兼容性**：使用`QFont()`默认构造函数替代`QFontDatabase::systemFont()`
- **字体回退**：当获取的字体为空时，使用默认字体（Noto Sans和Monospace）
- **向后兼容**：保持API接口不变，确保其他代码无需修改

### 3. 其他相关修复建议
1. **系统字体配置**：确保系统安装了中文字体包
   ```bash
   sudo apt install fonts-noto-cjk fonts-noto-cjk-extra
   ```

2. **环境变量设置**：在启动脚本中设置正确的locale环境变量
   ```bash
   export LANG=zh_CN.UTF-8
   export LANGUAGE=zh_CN:zh
   export LC_ALL=zh_CN.UTF-8
   ```

3. **字体缓存更新**：构建后更新字体缓存
   ```bash
   sudo fc-cache -fv
   ```
